# 编程策略与常用写法

## 规划优先级
- 优先满足科目完成条件。
- 其次满足轨迹闭合要求。
- 再满足定点起降。
- 同时尽量降低碰撞风险与不必要的飞行器数量。

## 距离与安全性
- 默认轨迹与杆件水平最近距离建议 `>=40cm`。
- 不应小于 `30cm`，低于该值碰撞风险显著上升。

## 科目1（绕竖杆）推荐写法
- 常用方形闭环绕行。
- 绕行前可设置 1 号/2 号电机灯光为绿色标识机头方向。
- 封闭绕行段建议交替结构：`转向/转动 -> 平移 -> 转向/转动 -> 平移`。
- 优先使用 `Goertek_TurnTo` 降低朝向误判概率。
- 进场/离场段若不影响科目判定，不要为套模板额外插入转向。

## 非科目1时的转向策略
- 除非用户明确要求机头朝向控制，且科目判定依赖该朝向，否则不要为套模板额外插入 `Goertek_Turn`。
- 非科目1优先以最少积木完成目标；若连续平移可满足判定，不要强行改成“转动+平移”交替。

## 参数缺失处理
- 对于由现场题卡决定的参数（例如特定高度），若用户未提供，不要臆造。
- 应明确提示“待现场参数”，并建议补充后再精修。
