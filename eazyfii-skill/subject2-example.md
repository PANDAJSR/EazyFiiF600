# 科目2示例（绕横杆）

> 仅用于示例说明，实际坐标、速度、延时需按现场与路线复检后调整。
> 关键词：`科目2`、`科目二`、`subject2`、`绕横杆`、`闭合`、`示例步骤`、`对应积木写法`。

## 示例前提
- 假设科目2两端支撑竖杆底座约为 `(240,80)`、`(320,80)`。
- 则横杆中点可近似取 `(280,80)`，示例按此中轴构造绕行。
- 示例目标：在 `X=280` 的截面上完成 `Y-Z` 平面的闭合矩形绕行。

## 变通说明（重点）
- 上述只是“横向排布”的示例：两端竖杆常见为 `y` 相同、`x` 不同。
- 实际题目也可能出现“看起来横杆是竖着的”排布：两端竖杆 `x` 相同、`y` 不同。
- 这种情况本质仍是绕“横杆轴”闭合，不是绕竖杆本体；只需要把示例思路做坐标轴映射。
- 可按下面方式变通：
- 原示例固定 `X=常量`，在 `Y-Z` 平面闭环。
- 若两杆为 `x` 相同、`y` 不同，则改为固定 `Y=常量`，在 `X-Z` 平面闭环。
- 例如可将点序列从 `(280,40,130) -> (280,120,130) -> (280,120,170) -> (280,40,170) -> (280,40,130)`，
- 映射为 `(240,80,130) -> (320,80,130) -> (320,80,170) -> (240,80,170) -> (240,80,130)`（仅示意）。

## 示例步骤（仅示例）
1. 智能平移到 `(280,40,130)`（绕行起点，位于横杆下方前侧）
2. 智能平移到 `(280,120,130)`（下方后侧）
3. 智能平移到 `(280,120,170)`（上方后侧）
4. 智能平移到 `(280,40,170)`（上方前侧）
5. 智能平移到 `(280,40,130)`（回到起点，完成科目2封闭图形）
6. 离开科目2区域（离场段位于封闭图形外，且科目2默认不要求机头朝向）

## 对应积木写法（字段示意）
- `EazyFii_MoveToCoordAutoDelay`：`{ X: "280", Y: "40", Z: "130", time: "800" }`
- `EazyFii_MoveToCoordAutoDelay`：`{ X: "280", Y: "120", Z: "130", time: "800" }`
- `EazyFii_MoveToCoordAutoDelay`：`{ X: "280", Y: "120", Z: "170", time: "800" }`
- `EazyFii_MoveToCoordAutoDelay`：`{ X: "280", Y: "40", Z: "170", time: "800" }`
- `EazyFii_MoveToCoordAutoDelay`：`{ X: "280", Y: "40", Z: "130", time: "800" }`

## 路径几何说明（对应你给的参考图）
- 路径点顺序：`(280,40,130) -> (280,120,130) -> (280,120,170) -> (280,40,170) -> (280,40,130)`。
- 等价理解：固定 `X=280`，在 `Y` 与 `Z` 方向分别做 `40~120`、`130~170` 的矩形闭环。
- 横杆位于两竖杆之间时，这条闭环可以作为“绕横杆一圈并闭合”的有效示例路径。
- 若障碍物排布旋转为“两杆 `x` 相同、`y` 不同”，则同理改为固定 `Y`，在 `X-Z` 平面完成等价闭环。

## 复检提醒
- 写入示例后，仍需调用轨迹问题复检工具确认科目2是否真正达成。
- 若闭合判定未通过，优先检查末段是否严格返回起点，以及中间是否被插入了非预期偏移点。
- 科目2示例默认不包含灯光积木，也不要求机头朝向飞行方向。
